### Why this was prioritized

AegisQuant has the most aggressive tech surface area of the 6 targets (RL + LLM consensus + live broker + Streamlit) but visible repo hygiene weakens the impression: `=0.2.36`, `=0.29.1`, `=2.3.0`, `pytest_clean.txt`, `pytest_output.txt`, `pytest_run.txt`, `pcr.txt`, and committed `.zip` model snapshots live at repo root. A reviewer landing on the file list before the README sees the noise first. Strengthening `.gitignore` plus a real CI workflow is the lowest-risk, highest-trust-signal pair of changes available without a local test loop.

### Evaluated and skipped

- **Delete the stray files.** Tempting but unsafe in a single commit: some scripts may grep for the `.zip` snapshots or the `pcr.txt` output, and there is no test-run feedback loop in this environment to verify a deletion is safe.
- **Anthropic-only LLM consensus layer.** README mentions LLM consensus scoring; promoting Claude to that role would align the project with the rest of the portfolio. Material refactor that deserves its own dedicated branch.
- **CI without dependent jobs.** The first iteration deliberately runs the syntax check as a fast gate before paying for the heavier `pytest` job, so the workflow fails cheaply on import errors.
